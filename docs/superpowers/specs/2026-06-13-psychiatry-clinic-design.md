# Спецификация: Сайт центра психиатрии «Гармония»

**Дата:** 2026-06-13
**Статус:** Утверждён к реализации
**Референс:** `REACT/doverie-clinic` (та же архитектура и дизайн-система)

## 1. Цель и контекст

Одностраничный лендинг для центра ментального здоровья **«Гармония»** — частная
психиатрическая практика врача-психотерапевта Шакарима Ануаровича (12 лет опыта).
Задача сайта: внушить доверие, снять стигму с обращения к психиатру, объяснить
форматы помощи (в т.ч. анонимные и выездные) и собрать заявки на консультацию.

Тон коммуникации: деликатный, без осуждения, акцент на анонимность,
конфиденциальность и человечность.

## 2. Технологический стек

Полностью повторяет референс `doverie-clinic`:

- **Next.js 16.2.7** (Pages Router) + React 19
- **JavaScript** (не TypeScript), alias `@/*`
- **Tailwind CSS v4** (`@tailwindcss/postcss`)
- **framer-motion 12** — анимации появления и микровзаимодействия
- **lucide-react** — иконки
- Контент централизован в `lib/site.js`
- Заявки уходят на `pages/api/lead.js`

## 3. Архитектура

Та же модель, что в референсе: единый контент-конфиг + тонкие секционные
компоненты + общие UI-примитивы.

```
psychiatry-clinic/
  lib/site.js              # ВЕСЬ контент: CLINIC, NAV, STATS, CONDITIONS,
                           # FORMATS, DOCTOR, STEPS, PRICING, TESTIMONIALS,
                           # FAQ, QUIZ, VALUES
  styles/globals.css       # дизайн-система: индиго-голубые CSS-токены + анимации
  components/
    ui/primitives.js       # Container, Section, Eyebrow, Button, Reveal, токены
    Navbar.js
    Hero.js
    Conditions.js          # «Заболевания, которые лечим»
    Formats.js             # «Форматы помощи»
    Doctor.js              # карточка одного врача (Шакарим Ануарович)
    Steps.js               # «Как проходит приём»
    Pricing.js
    Testimonials.js
    Faq.js
    QuizForm.js            # анонимный тест-самооценка
    Contacts.js            # форма заявки
    Footer.js
    FloatingCTA.js
  pages/
    _app.js, _document.js  # подключение шрифтов (Lexend + Source Sans 3)
    index.js               # сборка секций
    api/lead.js            # приём заявок (лог + 200, как в референсе)
  public/
    doctor-shakarim.jpg    # Фото1 (кладёт пользователь; есть fallback на инициалы)
    favicon.ico
```

**Принцип изоляции:** каждый компонент рисует одну секцию, читает данные из
`lib/site.js`, не знает о других секциях. Поменять текст/цену = править только
`lib/site.js`. Поменять палитру = править только `globals.css`.

## 4. Дизайн-система (палитра «Спокойный индиго-голубой»)

Светлая тема, WCAG-friendly (текстовые пары ≥4.5:1). Переопределяются
CSS-переменные в `:root` и `@theme inline`; структура анимаций (aurora, orb,
mesh, reveal, glow-ring, grid-pattern, glass) переносится из референса как есть,
меняются только цвета свечений на индиго/голубой.

Ориентировочные токены (финализируются через ui-ux-pro-max на этапе реализации):

| Токен | Значение | Назначение |
|-------|----------|-----------|
| `--background` | `#F7F9FC` | холодный off-white |
| `--foreground` | `#1E293B` | глубокий slate — основной текст |
| `--primary` | `#4F46E5` | индиго — бренд |
| `--primary-600/700` | `#4338CA / #3730A3` | заливки/hover (белый текст ≥4.5:1) |
| `--secondary` | `#6366F1` | светлее индиго |
| `--accent` | `#0EA5E9` | небесно-голубой — CTA |
| `--warm` | `#F59E0B` | звёзды/выделения |
| `--muted` | `#EEF2FB` | мягкая indigo-gray поверхность |
| `--card` | `#FFFFFF` | карточки |
| `--border` | `#DCE3F0` | бордеры с индиго-оттенком |

Шрифты: **Lexend** (заголовки) + **Source Sans 3** (текст) — как в референсе.
Поддержка `prefers-reduced-motion`, видимый focus-ring, skip-link.

## 5. Контент секций

### Hero
Заголовок про деликатную психиатрическую помощь, подзаголовок, статистика
(`STATS`: 12+ лет опыта · анонимно · выезд на дом · 24/7), два CTA — «Позвонить» и
«Анонимная консультация». Индиго-aurora фон.

### Заболевания, которые лечим (`CONDITIONS`)
Сетка карточек с иконками lucide:
Шизофрения · Деменция · Болезнь Альцгеймера · Болезнь Паркинсона · Депрессия ·
Панические атаки · ОКР · Биполярное расстройство · Делирий · Острый психоз.
Каждая карточка: иконка, название, 1 строка пояснения.

### Форматы помощи (`FORMATS`)
Карточки с описанием формата:
- Консультация психиатра
- Консультация психотерапевта
- Психиатр на дому (выезд)
- Ведение пациента с психическими заболеваниями
- Психотерапия и сопровождение
- Лечение в стационаре
- Выездная анонимная консультация

### Врач (`DOCTOR`)
Одна крупная карточка: фото (`/doctor-shakarim.jpg`, fallback — инициалы «ША» на
индиго-градиенте), имя «Шакарим Ануарович», роль «Врач-психиатр, психотерапевт»,
«12 лет опыта», теги специализаций, краткое описание подхода.

### Как проходит приём (`STEPS`)
4 шага: 1) Звонок/заявка → 2) Диагностика и беседа → 3) Индивидуальный план
лечения → 4) Сопровождение и поддержка.

### Цены (`PRICING`)
Заглушки «от …» (пользователь заменит): первичная консультация, консультация
психотерапевта, выезд на дом, ведение пациента, лечение в стационаре, выездная
анонимная консультация. Перки: анонимность, без постановки на учёт без согласия,
фиксированная цена, рассрочка.

### Отзывы (`TESTIMONIALS`)
3–5 анонимизированных отзывов (имена сокращены ради приватности, как в референсе),
рейтинг 5 звёзд.

### FAQ (`FAQ`)
Анонимно ли это? · Ставят ли на учёт? · Возможен ли выезд на дом? · Что такое
стационар и когда он нужен? · Сколько стоит?

### Анонимный тест-самооценка (`QUIZ`)
5 вопросов про настроение / тревогу / сон / интерес к жизни / для кого тест.
Каждый ответ 0..3 балла, 3 итоговых диапазона результата с мягкими формулировками.
**Дисклеймер обязателен:** «Результат не является диагнозом; при тревожных
признаках обратитесь к специалисту».

### Контакты (`Contacts`)
Форма заявки (имя, телефон, согласие) → `POST /api/lead`. Рядом — телефон,
WhatsApp, адрес, режим работы (заглушки).

### Footer + FloatingCTA
Подвал с контактами и дисклеймером; на мобильных — липкая панель
«Позвонить / WhatsApp».

## 7. Обработка ошибок и крайние случаи

- **Фото врача отсутствует** → карточка показывает инициалы «ША» на градиенте
  (тот же graceful-fallback, что в `Doctors.js` референса).
- **Форма заявки** → клиентская валидация (имя + телефон), состояние
  loading/success/error, как в `QuizForm`/`Contacts` референса; API всегда
  отвечает 200 и логирует (реальной интеграции CRM нет — вне scope).
- **prefers-reduced-motion** → все анимации отключаются.
- **Тест** → всегда показывает дисклеймер, не сохраняет ответы на сервер.

## 8. Использование инструментов

- **ui-ux-pro-max** — финализация палитры/типографики/иерархии секций и
  ревью UI на этапе реализации.
- **framer-motion** — `Reveal`-обёртки, stagger-сетки, micro-interactions
  кнопок (перенос из примитивов референса).
- **21st.dev / magic** — источник идей для карточек заболеваний и hero
  (вдохновение, не прямая вставка; код адаптируется под дизайн-систему).

## 9. Вне scope (YAGNI)

- Мультиязычность (только русский).
- Реальная CRM/email-интеграция формы (только лог, как в референсе).
- Личные кабинеты, онлайн-оплата, запись в календарь.
- Блог/статьи, отдельные страницы (одностраничный лендинг).
- Тёмная тема.

## 10. Критерии готовности

- `npm run dev` поднимается без ошибок, `npm run build` проходит.
- Все 12 секций рендерятся, контент из `lib/site.js`.
- Палитра индиго-голубая применена, контраст ≥4.5:1.
- Анимации работают и отключаются при reduced-motion.
- Форма заявки шлёт `POST /api/lead` и показывает success-состояние.
- Адаптив: мобайл/планшет/десктоп; FloatingCTA только на мобильных.
- Фото врача с fallback на инициалы.
